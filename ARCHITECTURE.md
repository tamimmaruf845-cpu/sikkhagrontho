## Key Architecture Principles

1. **Separation of Concerns**
   - Authentication logic isolated in `auth.js`
   - Data management in `data.js`
   - UI rendering in `main.js`
   - Admin functions in `admin.js`

2. **Role-Based Access**
   - Two distinct user roles: Admin and Student
   - Protected routes with validation
   - Dynamic UI based on user role

3. **Client-Side Persistence**
   - All data stored in localStorage
   - No backend required
   - Instant updates and persistence

4. **Modular Design**
   - Reusable functions
   - Event-driven architecture
   - Clean separation between pages

5. **Security First**
   - Role validation on every protected page
   - Session management
   - Data isolation between users
   - Admin account protection

6. **User Experience**
   - Smooth animations
   - Responsive design
   - Intuitive navigation
   - Glassmorphic aesthetics
   - Theme customization
